## 2. Fix Verification

### 2.1 Fix #1: CredentialsProvider + JWT Strategy

- **Before**: `session: { strategy: 'database' }` -- CredentialsProvider와 비호환
- **After**: `session: { strategy: 'jwt' }` (`[...nextauth].js:133-135`)
- **Verification**: jwt 콜백(`line 103-121`)에서 token에 user 정보 로드, session 콜백(`line 122-131`)에서 token 기반으로 session 구성
- **Result**: CONFIRMED -- jwt strategy 전환 완료, jwt/session 콜백 모두 정상 구현

### 2.2 Fix #2: PATCH 응답 Password 제외

- **Before**: `updateUserById` 결과 전체 User 객체 반환 (password 해시 포함 가능)
- **After**: `const { password: _pw, ...safeResult } = result` 후 `safeResult` 반환 (`admin/members/[id]/route.ts:17-18`)
- **Result**: CONFIRMED -- password 필드가 응답에서 제외됨

### 2.3 Fix #3: console.log 제거

- **Before**: `app/api/members/[id]/route.ts` 에 `console.log(id)`, `console.log(body)` 잔존
- **After**: 해당 파일에 console.log 없음
- **Result**: CONFIRMED -- 디버그 로그 완전 제거됨

### 2.4 Fix #4: 활성 회원 등급변경 버튼 추가

- **Before**: 활성 회원 탭에 등급 변경 UI 없음
- **After**: `handleLevelChange` 함수 구현(`line 43-52`), [등급변경] Button 렌더링(`line 148-155`)
- **Behavior**: prompt로 새 등급 입력 -> PATCH /api/admin/members/[id] 호출 -> mutate로 목록 갱신
- **Result**: CONFIRMED -- 설계의 [등급변경] 기능 구현 완료
