# Plan: 회원가입

> 작성일: 2026-03-10
> 기능명: 회원가입 (Member Registration Flow)
> 프로젝트 레벨: Dynamic

---

## 1. 배경 및 목적

### 현재 상황
- 카카오/네이버 소셜 로그인으로 OAuth 자동 가입 → `level=1` 즉시 부여
- 신규 가입 시 추가 프로필 입력 단계 없음
- 관리자가 수동으로 회원 등급 관리

### 문제점
- 처음 로그인하는 사용자가 바로 `level=1` 회원으로 활성화됨 (관리자 승인 없이)
- 소셜에서 가져오는 이름/성별/연락처 정보가 불완전할 수 있음
- 관리자가 신규 가입자를 파악하고 승인하는 체계가 없음

### 목표
1. 소셜 로그인 후 **프로필 완성 페이지** 로 리디렉션 (최초 1회)
2. **이메일/패스워드 회원가입** 추가 (테스트 및 소셜 계정이 없는 사용자용)
3. 신규 가입자는 `level=0` (대기) → 관리자 승인 후 `level=1` 활성화
4. 관리자 페이지에서 **가입 신청자 목록 조회 및 승인/거절** 기능

---

## 2. 기능 요구사항

### FR-01: 최초 로그인 후 프로필 완성
- 소셜 로그인 성공 시 `level=0` 으로 생성
- 프로필 미완성 사용자(`level=0`)는 `/register` 페이지로 리디렉션
- `/register` 에서 수집하는 정보:
  - 이름 (소셜에서 가져온 값 기본 표시, 수정 가능)
  - 성별 (남성/여성)
  - 연락처 (선택)
- 프로필 제출 후 `level=0` 유지 (관리자 승인 대기)

### FR-02: 관리자 가입 승인 페이지
- `/admin/members` 페이지 (level >= 4 이상만 접근)
- 탭 구성:
  - **가입 대기** (`level=0`): 이름, 가입일, 성별, 연락처 표시 + 승인/거절 버튼
  - **활성 회원** (`level=1,2,3`): 전체 회원 목록 + 등급 변경
- 승인 시 `level=1` 업데이트
- 거절 시 계정 삭제 (또는 level=-1 처리)

### FR-03: 미완성 프로필 접근 제한
- `level=0` 인 사용자가 `/register` 외 페이지 접근 시 `/register` 로 리디렉션
- 비로그인 사용자는 기존 동작 유지 (공개 페이지 접근 가능)

### FR-04: 가입 완료 대기 안내 화면
- 프로필 제출 후 "관리자 승인 대기 중" 안내 페이지 표시
- 승인 후 로그인 시 정상 접근 가능

### FR-05: 이메일/패스워드 회원가입
- 로그인 페이지(`/auth/signin`)에 이메일 회원가입 옵션 추가
- 가입 정보: 이메일, 패스워드, 이름, 성별
- 패스워드 해싱 저장 (`bcryptjs`)
- NextAuth `CredentialsProvider` 로 이메일 로그인 지원
- 가입 후 `level=0` (대기) → 관리자 승인 후 활성화
- User 모델에 `password` 필드 추가 필요 (Prisma 마이그레이션)

---

## 3. 비기능 요구사항

- 소셜 로그인 기존 플로우 (카카오/네이버) 변경 최소화
- 반응형 UI (모바일 우선)
- 기존 마이그레이션된 회원 데이터 영향 없음 (level >= 1 인 기존 회원은 그대로)

---

## 4. 현재 코드 분석

### 인증 흐름 (`pages/api/auth/[...nextauth].js`)
- `signIn` 콜백: 신규 유저 → `level=1` 로 `update` 처리
- **변경 필요**: `level=1` → `level=0` 으로 변경 (대기 상태)

### User 모델 (`prisma/schema.prisma`)
- `level Int @default(0)` ← 이미 0이 기본값
- `name`, `gender`, `phoneNumber` 필드 존재

### 기존 레벨 체계
| level | 의미 |
|-------|------|
| 0 | 신규 가입 (관리자 승인 대기) |
| 1 | 일반 회원 |
| 2 | 일반 회원 (상위) |
| 3 | 운영진 |
| 4+ | 관리자 |

## 6. 의존성

- NextAuth.js (기존) + `CredentialsProvider` 추가
- Prisma + PostgreSQL (기존) + `password` 필드 마이그레이션
- `bcryptjs` 패키지 설치 필요
- `useAuthRedirect` 훅 수정 필요
- `withSessionUser` 미들웨어 수정 필요

---

## 7. 성공 기준

1. 신규 소셜 로그인 → 프로필 입력 페이지 리디렉션
2. 이메일 회원가입 → 가입 후 대기 화면 표시
3. 이메일/패스워드로 로그인 가능
4. 관리자가 승인 → 해당 회원 정상 접근 가능
5. 기존 회원(level >= 1) 기능 영향 없음
