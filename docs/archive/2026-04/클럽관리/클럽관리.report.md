# Completion Report: 클럽 관리 시스템

> 기능명: 클럽 관리 (Club Management)
> 완료일: 2026-04-28
> PDCA Cycle: Plan → Design → Do → Check → Act → Report

---

## 1. 개요

Rallify 앱에 클럽 관리 시스템을 도입하여, 테니스 동호회를 생성하고 회원을 관리할 수 있는 기능을 구현했습니다. 기존 `level` 정수 기반 권한 체계를 `role` 기반 이중 구조(글로벌 + 클럽)로 전환하고, 이용약관/개인정보처리방침/회원탈퇴 기능도 함께 추가했습니다.

---

## 2. 구현 범위

### 2.1 역할 체계 전환

| 항목 | Before | After |
|------|--------|-------|
| 권한 모델 | `level` 정수 (0~5) | `role` enum (SUPER_ADMIN, USER, PENDING) |
| 인증 체크 | `level >= 1` | `role !== 'PENDING'` |
| 관리자 체크 | `level >= 4` | `role === 'SUPER_ADMIN'` |
| 클럽 역할 | 없음 | ClubMember.role (OWNER, MANAGER, MEMBER) |

데이터 마이그레이션: USER 28명, PENDING 13명, SUPER_ADMIN 2명

### 2.2 새 DB 모델

- **Club**: 이름, 설명, 지역, 가입방식(OPEN/APPROVAL/INVITE_ONLY), 소프트삭제
- **ClubMember**: 유저-클럽 관계, 역할, 상태(PENDING/INVITED/ACTIVE/LEFT/REMOVED/REJECTED)
- **User 확장**: `role`, `agreedTermsAt` 필드 추가

### 2.3 구현된 기능

| 기능 | API | 페이지 | 상태 |
