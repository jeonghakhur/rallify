# Plan: 클럽 관리 시스템

> 작성일: 2026-04-23
> 기능명: 클럽 관리 (Club Management)
> 프로젝트 레벨: Dynamic
> 참조: tennis-tab 설계서

---

## 1. 배경 및 목적

### 현재 상황
- 단일 앱에서 모든 회원이 동일한 공간을 공유
- `level` 정수값(0~5)으로 권한 관리: 0=대기, 1=일반, 3=운영진, 4+=관리자
- 관리자가 수동으로 회원 승인/등급 변경
- "클럽" 개념 없이 전체 사용자가 하나의 그룹

### 문제점
- 여러 테니스 클럽을 하나의 앱으로 관리할 수 없음
- 클럽별 자체 관리(회원 승인, 초대)가 불가능
- `level` 정수 기반 권한은 글로벌 역할과 클럽 역할을 구분할 수 없음

### 목표
1. **역할 체계 개편**: 글로벌 역할(시스템) + 클럽 역할(클럽 내) 이중 구조
2. **클럽 생성**: 일반회원이 클럽을 만들면 해당 클럽의 관리자가 됨
3. **가입/초대**: 클럽별 가입 방식(자유/승인/초대전용) 지원
4. **기존 기능 유지**: Schedule, GameResult 등 기존 기능은 변경하지 않음 (향후 클럽 스코프로 확장 가능)

## 2. 역할 체계

### 2.1 글로벌 역할 (Global Role)

시스템 전체에서의 사용자 역할. User 모델에 `role` 필드로 관리.

| 역할 | 값 | 설명 |
|------|-----|------|
| SUPER_ADMIN | `SUPER_ADMIN` | 전체 시스템 관리, 모든 클럽 접근 가능 |
| USER | `USER` | 일반회원 (소셜 로그인 시 기본값) |
| PENDING | `PENDING` | 소셜 로그인 후 프로필 미완성 (기존 level=0 대체) |

> 기존 level 0→PENDING, 1~3→USER, 4~5→SUPER_ADMIN 으로 마이그레이션

### 2.2 클럽 역할 (Club Role)

클럽 내에서의 멤버 역할. ClubMember 모델에 `role` 필드로 관리.

| 역할 | 값 | 권한 |
|------|-----|------|
| OWNER | `OWNER` | 클럽 설정 변경, 멤버 관리, 클럽 삭제 |
| MANAGER | `MANAGER` | 멤버 승인/거절, 초대, 일정 관리 |
| MEMBER | `MEMBER` | 일정 참여, 게임 결과 등록 |

### 2.3 현재 level과의 매핑

| 현재 level | 새 글로벌 역할 | 비고 |
|:----------:|:--------------:|------|
| 0 | PENDING | 프로필 미완성/승인 대기 |
| 1, 2 | USER | 일반회원 |
| 3 | USER | 운영진 → 클럽 MANAGER로 전환 |
| 4, 5 | SUPER_ADMIN | 시스템 관리자 |

## 3. 기능 요구사항

### FR-01: 클럽 생성
- 글로벌 역할이 USER인 사용자만 클럽 생성 가능
- 생성 시 입력 정보: 클럽 이름, 설명, 지역, 가입 방식
- 가입 방식(joinType): `OPEN`(자유 가입) / `APPROVAL`(승인 필요) / `INVITE_ONLY`(초대 전용)
- 생성자는 자동으로 해당 클럽의 `OWNER` 역할 부여
- 한 사용자가 여러 클럽 생성/가입 가능

### FR-02: 클럽 가입 신청
- `OPEN`: 즉시 ACTIVE 상태로 가입
- `APPROVAL`: PENDING 상태 → OWNER/MANAGER 승인 필요
- `INVITE_ONLY`: 직접 가입 불가, 초대만 가능
- 가입 시 자기소개(introduction) 입력 (선택)

### FR-03: 멤버 초대
- OWNER/MANAGER가 기존 앱 사용자를 클럽에 초대
- 초대받은 사용자는 INVITED 상태 → 수락/거절 선택
- 수락 시 ACTIVE, 거절 시 레코드 삭제

### FR-04: 멤버 승인/거절
- OWNER/MANAGER가 PENDING 상태 멤버를 승인/거절
- 승인 → ACTIVE, 거절 → REJECTED (사유 입력 가능)

### FR-05: 멤버 관리
- OWNER: 멤버 역할 변경 (MEMBER ↔ MANAGER), 멤버 강퇴
- MANAGER: 멤버 승인/거절, 초대
- MEMBER: 자발적 탈퇴
- OWNER는 탈퇴 불가 (소유권 이전 후에만 가능)

### FR-06: 클럽 목록/상세
- 전체 클럽 목록 조회 (공개 정보)
- 클럽 상세: 멤버 수, 가입 방식, 설명
- 내 클럽 목록 (가입한 클럽들)

### FR-07: 이용약관 및 개인정보처리방침
- `/terms` 페이지: 서비스 이용약관 (회원 가입/탈퇴, 서비스 이용, 금지행위 등)
- `/privacy` 페이지: 개인정보처리방침 (수집 항목, 이용 목적, 보유 기간, 파기 절차 등)
- 회원가입 페이지(`/auth/signup`)에 약관 동의 안내 문구 + 링크 표시
  - "가입하면 이용약관과 개인정보처리방침에 동의하는 것으로 간주됩니다"
  - 이용약관, 개인정보처리방침 각각 링크
- 소셜 로그인 최초 프로필 완성(`/register`) 시에도 동일 문구 표시

### FR-08: 회원탈퇴
- 프로필 설정 페이지에 "회원탈퇴" 섹션 (Danger Zone)
- **탈퇴 차단 조건**:
  - 소유한 클럽이 있는 경우 → "클럽을 삭제하거나 소유권을 이전해주세요" 안내
- **탈퇴 확인 절차**:
  - 경고 모달: "탈퇴 시 모든 데이터가 삭제되며 복구할 수 없습니다"
  - 이메일 재입력으로 본인 확인
- **탈퇴 처리**:
  - ClubMember 레코드에서 status = 'LEFT' 처리
  - User 레코드 삭제 (hard delete) + CASCADE로 관련 데이터 정리
  - NextAuth 세션 종료 → 로그인 페이지로 리디렉션

### FR-09: 기존 관리 기능 전환
- 기존 `/admin/members` (글로벌 관리자 전용)는 SUPER_ADMIN만 접근
- 클럽별 멤버 관리는 클럽 관리 페이지에서 처리
- 기존 level 기반 접근 제어를 role 기반으로 전환

### FR-10: 클럽 소프트 삭제 및 재활성화
- OWNER가 클럽 삭제 시 `isActive = false` (소프트 삭제)
- 소프트 삭제된 클럽은 목록에서 숨김, 30일 후 자동 영구 삭제 (또는 수동)
- OWNER 또는 SUPER_ADMIN이 재활성화 가능
- SUPER_ADMIN만 영구 삭제(hard delete) 가능

## 4. DB 스키마 변경

### 4.1 새 테이블

```prisma
model Club {
  id              String       @id @default(cuid())
  name            String
  description     String?
  city            String?
  district        String?
  joinType        String       @default("APPROVAL")  // OPEN, APPROVAL, INVITE_ONLY
  maxMembers      Int?
  isActive        Boolean      @default(true)
  createdBy       String
  creator         User         @relation("ClubCreator", fields: [createdBy], references: [id])
  createdAt       DateTime     @default(now())
  updatedAt       DateTime     @updatedAt
  members         ClubMember[]
}

model ClubMember {
  id           String   @id @default(cuid())
  clubId       String
  club         Club     @relation(fields: [clubId], references: [id], onDelete: Cascade)
  userId       String
  user         User     @relation(fields: [userId], references: [id])
  role         String   @default("MEMBER")    // OWNER, MANAGER, MEMBER
  status       String   @default("PENDING")   // PENDING, INVITED, ACTIVE, LEFT, REMOVED, REJECTED
  introduction String?
  invitedBy    String?
  joinedAt     DateTime?
  createdAt    DateTime @default(now())

  @@unique([clubId, userId])
}
```

### 4.2 User 모델 변경

```prisma
model User {
  // 기존 필드 유지
  level    Int      @default(0)    // 레거시, 마이그레이션 후 제거
  role     String   @default("USER")  // 새 필드: SUPER_ADMIN, USER, PENDING

  // 새 관계
  createdClubs  Club[]        @relation("ClubCreator")
  clubMembers   ClubMember[]
}
```

### 4.3 데이터 마이그레이션

```sql
-- level → role 매핑
UPDATE "User" SET role = 'PENDING' WHERE level = 0;
UPDATE "User" SET role = 'USER' WHERE level IN (1, 2, 3);
UPDATE "User" SET role = 'SUPER_ADMIN' WHERE level >= 4;
```

## 7. 인증 플로우 변경

### 소셜 로그인 (신규 사용자)
```
소셜 로그인 → role=PENDING 생성 → /register 프로필 완성 → role=USER 전환
→ 클럽 가입/생성 선택
```

### 기존 level 기반 → role 기반 전환

| 기존 코드 | 변경 후 |
|-----------|---------|
| `level === 0` → /register 리디렉션 | `role === 'PENDING'` → /register 리디렉션 |
| `level >= 1` → 인증 통과 | `role !== 'PENDING'` → 인증 통과 |
| `level >= 4` → 관리자 | `role === 'SUPER_ADMIN'` → 시스템 관리자 |
| 클럽 관리 (없음) | `clubMember.role === 'OWNER' \|\| 'MANAGER'` |

## 10. 성공 기준

1. 소셜 로그인 → role=PENDING → 프로필 완성 → role=USER 전환 정상 작동
2. USER가 클럽 생성 → OWNER 역할 자동 부여
3. 클럽 가입 방식(OPEN/APPROVAL/INVITE_ONLY)별 플로우 정상 작동
4. OWNER/MANAGER가 멤버 승인/거절/초대/강퇴 가능
5. SUPER_ADMIN만 시스템 전체 사용자 관리 가능
6. 기존 사용자 데이터(level) → role 마이그레이션 무결성
7. 기존 Schedule/GameResult 기능에 영향 없음
8. 이용약관/개인정보처리방침 페이지 접근 가능
9. 회원가입/프로필 완성 시 약관 동의 안내 표시
10. 회원탈퇴 시 클럽 소유 차단 → 확인 모달 → 계정 삭제 → 세션 종료
11. 클럽 소프트 삭제/재활성화 정상 작동

## 11. tennis-tab 참조 사항

tennis-tab에서 차용하는 설계:
- **이중 역할 구조**: 글로벌 role + 클럽 role (동일)
- **ClubMember 상태 머신**: PENDING → ACTIVE, INVITED → ACTIVE/REMOVED
- **가입 방식 enum**: OPEN / APPROVAL / INVITE_ONLY

tennis-tab과 다른 점:
- **글로벌 역할 간소화**: 5단계(SUPER_ADMIN~RESTRICTED) → 3단계(SUPER_ADMIN, USER, PENDING)
- **협회(Association) 미포함**: 불필요
- **비등록 회원(is_registered=false) 미포함**: 모든 멤버는 앱 사용자
- **대회/레슨/커뮤니티 미포함**: Rallify 범위 아님
