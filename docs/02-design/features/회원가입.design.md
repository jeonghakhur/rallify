# Design: 회원가입

> 작성일: 2026-03-10
> 기능명: 회원가입 (Member Registration Flow)
> 참조 플랜: docs/01-plan/features/회원가입.plan.md

## 1. 전체 플로우

### 1-A. 소셜 로그인 신규 가입 플로우
```
[로그인 페이지] → 카카오/네이버 클릭
    → OAuth 콜백 → DB에 level=0으로 생성
    → /register (프로필 완성 페이지)
        → 이름/성별 입력 후 제출
        → PATCH /api/members/[id]
    → /register/pending (승인 대기 안내)
    → 관리자 승인 후 → 정상 접근
```

### 1-B. 이메일 회원가입 플로우
```
[로그인 페이지] → "이메일로 가입" 클릭
    → /auth/signup (이메일 가입 페이지)
        → 이메일/패스워드/이름/성별 입력
        → POST /api/auth/register
        → 계정 생성 (level=0, password 해싱)
        → CredentialsProvider로 자동 로그인
    → /register/pending (승인 대기 안내)
    → 관리자 승인 후 → 정상 접근
```

### 1-C. 관리자 승인 플로우
```
[관리자] /admin/members
    → "가입 대기" 탭에서 level=0 회원 목록 확인
    → "승인" 클릭 → PATCH /api/admin/members/[id] { level: 1 }
    → "거절" 클릭 → DELETE /api/admin/members/[id]
```

## 3. API 설계

### 3-1. `POST /api/auth/register` (신규)
이메일 회원가입

**Request Body:**
```json
{
  "email": "user@example.com",
  "password": "plaintext",
  "name": "홍길동",
  "gender": "남성"
}
```

**처리 로직:**
1. 이메일 중복 확인
2. `bcryptjs`로 password 해싱 (saltRounds: 10)
3. User 생성: `level=0`, `provider='email'`, `username=email앞부분`
4. 성공 시 `{ success: true }` 반환

**Response:**
- `201`: `{ success: true, userId }`
- `400`: `{ error: '이미 사용 중인 이메일입니다.' }`
- `422`: `{ error: '입력값 오류' }`

---

### 3-2. `GET /api/admin/members` (신규)
관리자용 회원 목록 (level >= 4 만 접근 가능)

**Query Params:**
- `?status=pending` → level=0 대기자 목록
- `?status=active` → level>=1 활성 회원 목록
- 없으면 전체

**Response:**
```json
[
  {
    "id": "cuid",
    "name": "홍길동",
    "email": "user@example.com",
    "gender": "남성",
    "level": 0,
    "provider": "kakao",
    "createdAt": "2026-03-10T..."
  }
]
```

---

### 3-3. `PATCH /api/admin/members/[id]` (신규)
회원 승인 (level 변경)

**Request Body:**
```json
{ "level": 1 }
```

**처리 로직:** `updateUserById(id, { level })` 호출

**Response:** `{ id, name, level }`

---

### 3-4. `DELETE /api/admin/members/[id]` (신규)
회원 거절 (계정 삭제)

**처리 로직:** User 레코드 삭제 (CASCADE로 Account/Session 함께 삭제)

**Response:** `{ success: true }`

---

### 3-5. 기존 `pages/api/auth/[...nextauth].js` 변경

**변경 1: CredentialsProvider 추가**
```js
import CredentialsProvider from 'next-auth/providers/credentials';
import bcrypt from 'bcryptjs';

CredentialsProvider({
  name: 'Email',
  credentials: {
    email: { label: 'Email', type: 'email' },
    password: { label: 'Password', type: 'password' },
  },
  async authorize(credentials) {
    const user = await prisma.user.findUnique({
      where: { email: credentials.email },
    });
    if (!user || !user.password) return null;
    const isValid = await bcrypt.compare(credentials.password, user.password);
    if (!isValid) return null;
    return { id: user.id, name: user.name, email: user.email };
  },
})
```

**변경 2: signIn 콜백 - level=0으로 변경**
```js
// 기존
level: 1,
// 변경
level: 0,
```

**변경 3: session 콜백 - name 추가**
```js
session.user.name = user.name ?? '';  // 추가 (프로필 완성 여부 판단용)
```

## 4. 페이지 설계

### 4-1. `/auth/signup` (신규)
이메일 회원가입 폼

**컴포넌트:** `app/auth/signup/page.tsx`

**UI 구성:**
```
[이메일 가입]
이메일: [________________]
패스워드: [________________]
이름: [________________]
성별: ( 남성  ) ( 여성  )
[가입하기]
[로그인으로 돌아가기]
```

**유효성 검사:**
- 이메일: 형식 검증
- 패스워드: 최소 8자
- 이름: 최소 2자
- 성별: 필수 선택

**처리 흐름:**
1. 폼 제출 → `POST /api/auth/register`
2. 성공 → `signIn('credentials', { email, password })` 호출
3. 로그인 성공 → `/register/pending` 리디렉션

---

### 4-2. `/register` (신규)
소셜 로그인 후 프로필 완성 페이지

**컴포넌트:** `app/register/page.tsx`

**접근 조건:** `level=0` && `name === null`

**UI 구성:**
```
[프로필을 완성해주세요]
이름: [소셜에서 가져온 이름 기본값]
성별: ( 남성  ) ( 여성  )
연락처: [________________] (선택)
[완료]
```

**처리 흐름:**
1. 폼 제출 → `PATCH /api/members/[userId]` `{ name, gender, phoneNumber }`
2. 성공 → `/register/pending` 리디렉션

---

### 4-3. `/register/pending` (신규)
가입 완료 대기 안내 화면

**컴포넌트:** `app/register/pending/page.tsx`

**접근 조건:** `level=0` && `name !== null`

**UI 구성:**
```
[가입 신청이 완료되었습니다]
관리자 승인 후 서비스를 이용하실 수 있습니다.
승인까지 1~2일 소요될 수 있습니다.

[로그아웃]
```

---

### 4-4. `/admin/members` (신규)
관리자 회원 관리 페이지

**컴포넌트:** `app/admin/members/page.tsx`

**접근 조건:** `level >= 4`

**UI 구성:**
```
[가입 대기 | 활성 회원]

[가입 대기 탭]
이름 | 이메일 | 성별 | 가입일 | 가입방법 | 액션
홍길동 | user@... | 남성 | 3/10 | 카카오 | [승인] [거절]

[활성 회원 탭]
이름 | 이메일 | 등급 | 성별 | 액션
김철수 | admin@... | 3 | 남성 | [등급변경]
```

## 5. 클라이언트 접근 제어

### `hooks/useAuthRedirect.tsx` 변경
```ts
export default function useAuthRedirect(redirectTo = '/auth/signin', minLevel = 0) {
  const { data: session, status } = useSession();
  const router = useRouter();
  const pathname = usePathname();

  useEffect(() => {
    if (status === 'loading') return;

    if (!session) {
      router.push(redirectTo);
      return;
    }

    const level = session.user.level;
    const name = session.user.name;

    // level=0 접근 제어
    if (level === 0) {
      if (!name && pathname !== '/register') {
        router.push('/register');
      } else if (name && pathname !== '/register/pending') {
        router.push('/register/pending');
      }
      return;
    }

    if (level < minLevel) {
      router.push(redirectTo);
    } else {
      setIsChecking(false);
    }
  }, [session, status, router, redirectTo, minLevel, pathname]);
}
```

## 6. 서비스 레이어 변경

### `service/user.ts` 추가 함수

```ts
// 이메일 회원 생성
export async function createEmailUser(data: {
  email: string;
  password: string;  // 이미 해싱된 값
  name: string;
  gender: string;
}) { ... }

// 관리자용 대기 회원 목록
export async function getPendingMembers() {
  return prisma.user.findMany({
    where: { level: 0 },
    orderBy: { createdAt: 'asc' },
  });
}

// 회원 삭제 (거절)
export async function deleteUserById(id: string) {
  return prisma.user.delete({ where: { id } });
}
```

## 9. 구현 순서

1. **Prisma 스키마** → `password` 필드 추가 + `migrate dev`
2. **패키지 설치** → `bcryptjs`
3. **서비스 레이어** → `createEmailUser`, `getPendingMembers`, `deleteUserById`
4. **NextAuth 수정** → CredentialsProvider 추가, level=0, session name
5. **API 라우트** → `/api/auth/register`, `/api/admin/members`, `/api/admin/members/[id]`
6. **페이지** → `/auth/signup`, `/register`, `/register/pending`, `/admin/members`
7. **훅 수정** → `useAuthRedirect` level=0 로직

## 10. 성공 기준 (테스트)

| 시나리오 | 기대 결과 |
|---------|-----------|
| 새 이메일로 가입 | `/register/pending` 로 이동 |
| 중복 이메일로 가입 시도 | 에러 메시지 표시 |
| 카카오 신규 로그인 | `/register` 로 이동 |
| `/register` 프로필 완성 | `/register/pending` 로 이동 |
| level=0 유저가 `/` 접근 | `/register/pending` 로 리디렉션 |
| 관리자 승인 | 해당 유저 level=1 업데이트 |
| 승인된 유저 로그인 | 정상 메인 페이지 접근 |
| 기존 회원(level>=1) 로그인 | 기존과 동일하게 동작 |
