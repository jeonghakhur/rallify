## 1. DB 스키마

### 1.1 User 모델 변경

```prisma
model User {
  id            String    @id @default(cuid())
  username      String?   @unique
  name          String?
  email         String?   @unique
  emailVerified DateTime?
  image         String?
  level         Int       @default(0)          // 레거시 (마이그레이션 후 제거 예정)
  role          String    @default("USER")     // NEW: SUPER_ADMIN, USER, PENDING
  gender        String?
  provider      String?
  phoneNumber   String?
  birthday      String?
  birthyear     String?
  address       String?
  password      String?
  agreedTermsAt DateTime?                      // NEW: 약관 동의 일시
  createdAt     DateTime  @default(now())

  // 기존 관계
  accounts         Account[]
  sessions         Session[]
  schedules        Schedule[]        @relation("ScheduleAuthor")
  attendees        Attendee[]
  scheduleComments ScheduleComment[]
  gameResults      GameResult[]      @relation("GameResultAuthor")
  gameComments     GameComment[]
  editHistories    EditHistory[]

  // 새 관계
  createdClubs     Club[]            @relation("ClubCreator")
  clubMembers      ClubMember[]
}
```

### 1.2 Club 모델 (신규)

```prisma
model Club {
  id          String   @id @default(cuid())
  name        String
  description String?
  city        String?
  district    String?
  joinType    String   @default("APPROVAL")  // OPEN, APPROVAL, INVITE_ONLY
  maxMembers  Int?
  isActive    Boolean  @default(true)
  createdBy   String
  creator     User     @relation("ClubCreator", fields: [createdBy], references: [id])
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  members     ClubMember[]
}
```

### 1.3 ClubMember 모델 (신규)

```prisma
model ClubMember {
  id           String    @id @default(cuid())
  clubId       String
  club         Club      @relation(fields: [clubId], references: [id], onDelete: Cascade)
  userId       String
  user         User      @relation(fields: [userId], references: [id], onDelete: Cascade)
  role         String    @default("MEMBER")   // OWNER, MANAGER, MEMBER
  status       String    @default("PENDING")  // PENDING, INVITED, ACTIVE, LEFT, REMOVED, REJECTED
  statusReason String?
  introduction String?
  invitedBy    String?
  joinedAt     DateTime?
  createdAt    DateTime  @default(now())

  @@unique([clubId, userId])
}
```

### 1.4 데이터 마이그레이션 SQL

```sql
-- Step 1: User 테이블에 role 컬럼 추가
ALTER TABLE "User" ADD COLUMN "role" TEXT NOT NULL DEFAULT 'USER';
ALTER TABLE "User" ADD COLUMN "agreedTermsAt" TIMESTAMP;

-- Step 2: level → role 매핑
UPDATE "User" SET role = 'PENDING' WHERE level = 0;
UPDATE "User" SET role = 'USER' WHERE level IN (1, 2, 3);
UPDATE "User" SET role = 'SUPER_ADMIN' WHERE level >= 4;

-- Step 3: Club 테이블 생성
CREATE TABLE "Club" (
  "id" TEXT NOT NULL,
  "name" TEXT NOT NULL,
  "description" TEXT,
  "city" TEXT,
  "district" TEXT,
  "joinType" TEXT NOT NULL DEFAULT 'APPROVAL',
  "maxMembers" INTEGER,
  "isActive" BOOLEAN NOT NULL DEFAULT true,
  "createdBy" TEXT NOT NULL,
  "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
  "updatedAt" TIMESTAMP(3) NOT NULL,
  CONSTRAINT "Club_pkey" PRIMARY KEY ("id"),
  CONSTRAINT "Club_createdBy_fkey" FOREIGN KEY ("createdBy") REFERENCES "User"("id") ON DELETE RESTRICT
);

-- Step 4: ClubMember 테이블 생성
CREATE TABLE "ClubMember" (
  "id" TEXT NOT NULL,
  "clubId" TEXT NOT NULL,
  "userId" TEXT NOT NULL,
  "role" TEXT NOT NULL DEFAULT 'MEMBER',
  "status" TEXT NOT NULL DEFAULT 'PENDING',
  "statusReason" TEXT,
  "introduction" TEXT,
  "invitedBy" TEXT,
  "joinedAt" TIMESTAMP(3),
  "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT "ClubMember_pkey" PRIMARY KEY ("id"),
  CONSTRAINT "ClubMember_clubId_fkey" FOREIGN KEY ("clubId") REFERENCES "Club"("id") ON DELETE CASCADE,
  CONSTRAINT "ClubMember_userId_fkey" FOREIGN KEY ("userId") REFERENCES "User"("id") ON DELETE CASCADE,
  CONSTRAINT "ClubMember_clubId_userId_key" UNIQUE ("clubId", "userId")
);
```

## 2. API 설계

### 2.1 클럽 CRUD

#### `POST /api/clubs` — 클럽 생성

```ts
// Request
{ name: string; description?: string; city?: string; district?: string; joinType: "OPEN" | "APPROVAL" | "INVITE_ONLY" }

// Response 201
{ id, name, description, city, district, joinType, createdBy, createdAt }

// Logic
1. withSessionUser (role !== 'PENDING')
2. prisma.club.create({ data: { ...body, createdBy: user.id } })
3. prisma.clubMember.create({ data: { clubId, userId: user.id, role: 'OWNER', status: 'ACTIVE', joinedAt: now() } })
4. return club
```

#### `GET /api/clubs` — 클럽 목록

```ts
// Response 200
[{ id, name, description, city, district, joinType, memberCount, createdAt }]

// Logic
1. withSessionUser
2. prisma.club.findMany({ where: { isActive: true }, include: { _count: { select: { members: { where: { status: 'ACTIVE' } } } } } })
```

#### `GET /api/clubs/[id]` — 클럽 상세

```ts
// Response 200
{ id, name, description, city, district, joinType, maxMembers, memberCount, creator, myMembership?, members (ACTIVE only) }

// Logic
1. withSessionUser
2. prisma.club.findUnique + 멤버 수 + 현재 사용자의 ClubMember 조회
```

#### `PATCH /api/clubs/[id]` — 클럽 수정

```ts
// Request
{ name?, description?, city?, district?, joinType?, maxMembers? }

// Auth: OWNER 또는 SUPER_ADMIN
// Logic: ClubMember에서 userId+clubId로 role='OWNER' 확인
```

#### `DELETE /api/clubs/[id]` — 클럽 소프트 삭제

```ts
// Auth: OWNER (소프트 삭제) 또는 SUPER_ADMIN (영구 삭제)
// OWNER: prisma.club.update({ where: { id }, data: { isActive: false } })
// SUPER_ADMIN: prisma.club.delete({ where: { id } })  // CASCADE로 ClubMember도 삭제
```

### 2.2 클럽 멤버십

#### `POST /api/clubs/[id]/join` — 가입 신청

```ts
// Request
{ introduction?: string }

// Logic
1. withSessionUser (role !== 'PENDING')
2. 연락처(phoneNumber) 필수 체크 → 없으면 400 "연락처 등록이 필요합니다"
3. club.joinType 확인:
   - OPEN → status='ACTIVE', joinedAt=now()
   - APPROVAL → status='PENDING'
   - INVITE_ONLY → 400 "초대를 통해서만 가입할 수 있습니다"
4. maxMembers 체크: ACTIVE 멤버 수 >= maxMembers → 400
5. 이미 가입한 경우 → 409
6. prisma.clubMember.create(...)
```

#### `POST /api/clubs/[id]/invite` — 멤버 초대

```ts
// Request
{ userId: string; role?: "MEMBER" | "MANAGER" }

// Auth: OWNER 또는 MANAGER
// Logic
1. 대상 사용자 존재 확인
2. 이미 멤버인지 확인 → 409
3. prisma.clubMember.create({ data: { clubId, userId, role: role || 'MEMBER', status: 'INVITED', invitedBy: user.id } })
```

#### `POST /api/clubs/[id]/members/[memberId]/respond` — 초대 수락/거절

```ts
// Request
{ accept: boolean }

// Auth: 초대받은 본인만
// Logic
1. ClubMember.userId === session.user.id 확인
2. status === 'INVITED' 확인
3. accept → status='ACTIVE', joinedAt=now()
4. !accept → prisma.clubMember.delete(...)
```

#### `PATCH /api/clubs/[id]/members/[memberId]` — 멤버 상태/역할 변경

```ts
// Request
{ status?: "ACTIVE" | "REMOVED" | "REJECTED"; role?: "MEMBER" | "MANAGER"; statusReason?: string }

// Auth: OWNER 또는 MANAGER (MANAGER는 역할 변경 불가)
// Logic
1. 승인: status='ACTIVE', joinedAt=now()
2. 거절: status='REJECTED', statusReason 저장
3. 역할변경: OWNER만 가능 (MEMBER ↔ MANAGER)
4. OWNER 역할은 변경 불가
```

#### `DELETE /api/clubs/[id]/members/[memberId]` — 강퇴 또는 자발적 탈퇴

```ts
// Auth: OWNER(강퇴) 또는 본인(탈퇴)
// Logic
1. 본인 탈퇴: userId === session.user.id → status='LEFT'
2. 강퇴: OWNER만 → status='REMOVED', statusReason 저장
3. OWNER는 탈퇴/강퇴 불가
```

### 2.3 회원탈퇴

#### `DELETE /api/me` — 회원탈퇴

```ts
// Request
{ confirmEmail: string }

// Logic
1. confirmEmail === session.user.email 확인
2. 차단 조건 체크:
   - ClubMember where userId=user.id AND role='OWNER' AND status='ACTIVE'
   - 있으면 400: "소유한 클럽이 있습니다. 클럽을 삭제하거나 소유권을 이전해주세요."
     + 소유 클럽 목록 반환
3. ClubMember 정리: status → 'LEFT'
4. User 삭제: prisma.user.delete({ where: { id } })  // CASCADE
5. Response: { success: true }
```

### 2.4 사용자 검색 (초대용)

#### `GET /api/users/search?q=keyword`

```ts
// Query: q (최소 2자)
// Response 200
[{ id, name, username, image }]  // 최대 20건

// Logic
1. withSessionUser
2. prisma.user.findMany({
     where: { role: { not: 'PENDING' }, OR: [{ name: contains }, { username: contains }, { email: contains }] },
     select: { id, name, username, image },
     take: 20
   })
```

### 3.1 새 파일: `service/club.ts`

```ts
// 클럽 CRUD
```

## 4. 인증/권한 변경

### 4.1 NextAuth 콜백 수정 (`pages/api/auth/[...nextauth].js`)

```ts
// signIn 콜백 변경
// Before: level: 0
// After: role: 'PENDING' (level은 레거시 호환용으로 0 유지)

// jwt 콜백 변경
// Before: token.level = dbUser.level
// After: token.role = dbUser.role (level도 함께 전달하되 deprecate)

// session 콜백 변경
// Before: session.user.level = token.level
// After: session.user.role = token.role
```

### 4.2 미들웨어 수정 (`util/session.ts`)

```ts
// Before
export async function withSessionUser(handler, minLevel = 1)
  // user.level < minLevel → 401

// After
export async function withSessionUser(handler, options?: { requiredRole?: string })
  // user.role === 'PENDING' → 401 (기본)
  // requiredRole === 'SUPER_ADMIN' && user.role !== 'SUPER_ADMIN' → 403
```

### 4.3 클라이언트 훅 수정 (`hooks/useAuthRedirect.tsx`)

```ts
// Before
export function useAuthRedirect(redirectTo, minLevel = 0)
  // level === 0, no name → /register
  // level === 0, has name → /register/pending
  // level < minLevel → redirectTo

// After
export function useAuthRedirect(redirectTo, requiredRole?: string)
  // role === 'PENDING', no name → /register
  // role === 'PENDING', has name → /register/pending
  // requiredRole === 'SUPER_ADMIN' && role !== 'SUPER_ADMIN' → redirectTo
```

### 4.4 세션 타입 확장

```ts
// types/next-auth.d.ts (또는 model/user.ts)
declare module "next-auth" {
  interface Session {
    user: {
      id: string
      name: string
      email: string
      image?: string
      role: string       // NEW
      level: number      // DEPRECATED
      gender: string
      userName: string
    }
  }
}
```

### 5.6 `/clubs/[id]/manage` — 클럽 관리


## 6. 구현 순서

### Phase 1: 스키마 + 인증 전환 (기반)
1. Prisma 스키마에 Club, ClubMember 모델 추가 + User.role 필드 추가
2. Supabase 마이그레이션 적용 (DDL + level→role 데이터 매핑)
3. NextAuth 콜백 수정 (role 기반)
4. `withSessionUser` 미들웨어 수정
5. `useAuthRedirect` 훅 수정
6. 세션 타입 확장

### Phase 2: 약관 + 회원탈퇴
7. `/terms` 페이지 생성
8. `/privacy` 페이지 생성
9. 회원가입/프로필 완성 페이지에 약관 동의 안내 추가
10. `DELETE /api/me` API + deleteAccount 서비스
11. 프로필 수정 페이지에 Danger Zone + 탈퇴 모달

### Phase 3: 클럽 CRUD
12. `service/club.ts` 생성
13. `POST/GET /api/clubs` API
14. `GET/PATCH/DELETE /api/clubs/[id]` API
15. `/clubs`, `/clubs/create`, `/clubs/[id]` 페이지

### Phase 4: 멤버십
16. `POST /api/clubs/[id]/join` 가입 신청 API
17. `POST /api/clubs/[id]/invite` 초대 API
18. `POST /api/clubs/[id]/members/[memberId]/respond` 초대 응답 API
19. `PATCH /api/clubs/[id]/members/[memberId]` 승인/거절/역할 변경 API
20. `DELETE /api/clubs/[id]/members/[memberId]` 강퇴/탈퇴 API
21. `GET /api/users/search` 사용자 검색 API
22. `GET /api/my/clubs` 내 클럽 목록 API

24. `/my/clubs` 내 클럽 목록 페이지

## 9. 보안 고려사항

1. **API 권한 체크**: 모든 클럽 관리 API에서 ClubMember.role 검증
2. **OWNER 보호**: OWNER 역할 변경/강퇴 불가
3. **탈퇴 차단**: 클럽 소유자는 탈퇴 불가 (소유권 이전 또는 클럽 삭제 필요)
4. **이메일 확인**: 회원탈퇴 시 이메일 재입력으로 본인 확인
5. **maxMembers 체크**: 가입 신청 시 정원 초과 방지
6. **연락처 필수**: 클럽 가입 시 phoneNumber 등록 여부 체크
7. **중복 가입 방지**: @@unique([clubId, userId]) 제약조건
